# Формат коммитов проекта

Все коммиты в этом репозитории оформляются **строго** по конвенции ниже.
Применяй её **всегда и автоматически**, даже без явной просьбы пользователя.

## Структура сообщения

```
<type>: <описание на русском>
```

- `<type>` — **только** одно из: `feat` / `fix` / `refactor` / `hotfix`
  (сам префикс — латиницей, в нижнем регистре).
- `<описание>` — на **русском языке**, кратко описывает, **что было сделано**.
- Описание **начинается с глагола в форме совершённого действия**
  (страдательное причастие / прошедшее время, средний род):
  `добавлено`, `обновлено`, `реализовано`, `исправлено`, `удалено`,
  `вынесено`, `настроено`, `переименовано`, `сделано` и т.п.
  Список не исчерпывающий — важна именно эта форма («что сделано»),
  а не повелительная («добавь») или инфинитив («добавить»).
- После префикса `<type>:` описание пишется со строчной буквы.

## Когда какой type

| type       | Когда использовать                                                     |
|------------|------------------------------------------------------------------------|
| `feat`     | Новая функциональность, возможность, фича, новый модуль/скилл           |
| `fix`      | Исправление бага в существующем коде                                    |
| `refactor` | Изменение кода без изменения поведения (чистка, реструктуризация)       |
| `hotfix`   | Срочное исправление критичной проблемы (обычно в проде)                 |

## Примеры

```
feat: добавлена авторизация через OAuth
feat: реализована страница профиля пользователя
fix: исправлено падение при пустом списке товаров
refactor: вынесена логика запросов в отдельный composable
hotfix: устранена утечка памяти в SSR-рендере
```

## Правила

1. Одна строка заголовка — коротко и по делу. При необходимости можно
   добавить тело коммита через пустую строку (тоже на русском).
2. Не использовать типы вне списка (`chore`, `docs`, `style`, `test` и пр.
   в этом проекте **не применяются** — подбирай ближайший из четырёх).
3. Не переходить на английский в описании.
4. Не использовать повелительное наклонение («добавь», «исправь») —
   только «что сделано» («добавлено», «исправлено»).

## Согласование названия перед коммитом

Перед созданием коммита **всегда предлагай пользователю несколько вариантов
названия** (обычно 2–3) на выбор и **жди его одобрения**. Не выполняй
`git commit`, пока пользователь не подтвердил конкретный вариант (или не
предложил свой). Варианты должны соответствовать конвенции выше.
